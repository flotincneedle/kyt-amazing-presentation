# Documentation

## Core Documents
| File | Purpose | Update frequency |
|------|---------|-----------------|
| `VISION.md` | Full project vision, requirements, context | Rarely (source of truth) |
| `RESEARCH.md` | Tool & tech research with recommendations | When new tools evaluated |
| `ARCHITECTURE.md` | Technical architecture, data flow, components | When architecture changes |
| `ROADMAP.md` | Phased plan with milestones and progress | Every session |

## Process Documents
| File | Purpose | Update frequency |
|------|---------|-----------------|
| `adr/` | Architecture Decision Records | When decisions made |
| `prompts/` | Cross-validation prompts for Gemini/ChatGPT | Before major milestones |

## Root-level Documents
| File | Purpose | Update frequency |
|------|---------|-----------------|
| `../README.md` | Project overview, setup instructions | When stack/structure changes |
| `../CHANGELOG.md` | All notable changes | Every session |
| `../HANDOFF.md` | Session continuity notes | End of every session |
| `../CONTEXT.md` | Original brainstorm (historical) | Never (archive) |
